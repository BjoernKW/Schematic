# Use Case: Copy ER Diagram Source

## Overview

**Use Case ID:** UC-008
**Use Case Name:** Copy ER Diagram Source
**Primary Actor:** Developer
**Goal:** Copy the Mermaid source of the ER diagram to the clipboard so that it can be embedded in architecture documentation
**Status:** Draft

## Preconditions

- Developer has navigated to the ER diagram view and a diagram is currently rendered (UC-007 in progress)
- The browser supports clipboard write access

## Main Success Scenario

1. Developer selects the copy action on the ER diagram view.
2. System writes the Mermaid diagram source to the browser clipboard.
3. System displays a confirmation indicator that the source has been copied.
4. Developer pastes the Mermaid source into their documentation tool.

## Alternative Flows

### A1: Clipboard Access Denied by Browser

**Trigger:** The browser denies clipboard write access (step 2)
**Flow:**

1. System displays an error message indicating that clipboard access was not granted.
2. System displays the raw Mermaid source in a text area so the developer can copy it manually.
3. Use case ends.

### A2: Diagram Source Is Empty

**Trigger:** The generated diagram has no content (e.g., schema has no tables) (step 2)
**Flow:**

1. System copies an empty or minimal Mermaid source to the clipboard.
2. System displays the confirmation indicator.
3. Use case ends.

## Postconditions

### Success Postconditions

- The Mermaid diagram source is available on the developer's clipboard
- No data in the database is modified

### Failure Postconditions

- The clipboard is not updated
- System displays an error message or falls back to showing the source in a selectable text area
- No data in the database is modified

## Business Rules

### BR-001: Source Matches Rendered Diagram

The copied Mermaid source must be identical to the source used to render the currently displayed diagram; it is not regenerated at copy time.

### BR-002: Read-Only Operation

Copying the diagram source is a read-only operation; no database structures or data are altered.

### BR-003: PostgreSQL Scope

This use case is only reachable when a PostgreSQL database is connected, as ER diagram generation is limited to PostgreSQL in the current release (see UC-007, BR-001).
