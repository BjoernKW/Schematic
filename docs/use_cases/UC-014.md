# Use Case: Configure Row Preview Limit

## Overview

**Use Case ID:** UC-014
**Use Case Name:** Configure Row Preview Limit
**Primary Actor:** Developer
**Goal:** Configure the maximum number of rows shown in the table row preview so that the UI suits the table sizes and data density of the host application
**Status:** Draft

## Preconditions

- Schematic is added as a Maven dependency in the host application (UC-001 completed)
- Developer has write access to the host application's `application.yml` (or equivalent configuration file)

## Main Success Scenario

1. Developer opens the host application's `application.yml`.
2. Developer sets `schematic.preview-row-limit` to the desired positive integer value.
3. Developer saves the configuration file.
4. Developer restarts the application.
5. System reads the configured value at startup and applies it as the row limit for all table previews.
6. Developer navigates to a table in the Schematic UI and previews its rows.
7. System fetches and displays up to the configured number of rows for that table.

## Alternative Flows

### A1: Property Not Configured

**Trigger:** `schematic.preview-row-limit` is absent from `application.yml` (step 5)
**Flow:**

1. System uses the default value of 10 rows.
2. Use case continues at step 6.

### A2: Value Is Zero or Negative

**Trigger:** Developer sets `schematic.preview-row-limit` to a value less than 1 (step 2)
**Flow:**

1. System fails to start and logs a configuration error indicating that the value must be a positive integer.
2. Developer corrects the value in `application.yml` and restarts.
3. Use case continues at step 5.

### A3: Value Is Very Large

**Trigger:** Developer sets `schematic.preview-row-limit` to a large value (e.g. 10 000) (step 5)
**Flow:**

1. System applies the configured limit without restriction.
2. When Developer previews a large table, the query may take longer and the rendered page may be slow to load.
3. Use case continues at step 7.

### A4: Table Has Fewer Rows Than the Limit

**Trigger:** The previewed table contains fewer rows than `schematic.preview-row-limit` (step 7)
**Flow:**

1. System returns all rows in the table.
2. System displays the full result without a truncation notice.
3. Use case ends.

## Postconditions

### Success Postconditions

- All table row previews display at most the configured number of rows
- The configured limit persists across application restarts for as long as the property remains set
- No data in the database is modified

### Failure Postconditions

- If the configured value is invalid, the application does not start and logs a descriptive error
- If the property is absent, the default limit of 10 is used and all previews remain functional

## Business Rules

### BR-001: Default Value

When `schematic.preview-row-limit` is not configured, the system uses a default of 10 rows, preserving the behaviour prior to this feature.

### BR-002: Minimum Value

The configured value must be a positive integer (minimum 1). Zero and negative values are rejected at startup.

### BR-003: Uniform Application

The row limit applies uniformly to all table previews. It is not configurable per table.

### BR-004: Preview Only

The row limit applies exclusively to the table row preview (UC-003). Ad-hoc SQL query results (UC-004) are not subject to this limit.

### BR-005: Restart Required

A change to `schematic.preview-row-limit` takes effect only after the application is restarted. Live reload of this property is not supported.
