# Use Case: Sort and Filter Tables

## Overview

**Use Case ID:** UC-010
**Use Case Name:** Sort and Filter Tables
**Primary Actor:** Developer
**Goal:** Locate relevant tables quickly in a large schema by sorting and filtering the table list by name
**Status:** Draft

## Preconditions

- Developer has navigated to the table list and it is displaying results (UC-002 in progress)
- The table list contains at least one table

## Main Success Scenario

1. Developer enters a filter term into the search field on the table list.
2. System updates the displayed table list to show only tables whose names match the filter term.
3. System updates the list without a full page reload.
4. Developer reviews the filtered results and selects the desired table.

## Alternative Flows

### A1: Sort Order Changed

**Trigger:** Developer selects a sort order (e.g., ascending or descending by name) instead of or in addition to filtering (step 1)
**Flow:**

1. Developer selects the desired sort order.
2. System reorders the displayed table list accordingly without a full page reload.
3. Use case continues at step 4.

### A2: No Tables Match the Filter

**Trigger:** The entered filter term does not match any table name (step 2)
**Flow:**

1. System displays an empty list with an appropriate no-results message.
2. Developer modifies or clears the filter term.
3. Use case continues at step 2.

### A3: Filter Cleared

**Trigger:** Developer removes all text from the filter field (step 1)
**Flow:**

1. System restores the full unfiltered table list in the current sort order.
2. Use case continues at step 4.

### A4: Filter and Sort Combined

**Trigger:** Developer applies both a filter term and a sort order
**Flow:**

1. System applies the filter first, then sorts the matching results.
2. System displays the filtered and sorted list without a full page reload.
3. Use case continues at step 4.

## Postconditions

### Success Postconditions

- The table list reflects the applied filter and sort criteria
- Developer can select any visible table to proceed with browsing its schema (UC-002)
- No data in the database is modified

### Failure Postconditions

- The table list is unchanged or shows an error state
- System displays an appropriate message
- No data in the database is modified

## Business Rules

### BR-001: Client-Side Interaction

Filter and sort operations update the displayed list without a full page reload, providing immediate feedback as the developer types.

### BR-002: Filter Scope

Filtering applies to table names only; column names and other schema attributes are not included in the filter match.

### BR-003: Read-Only Operation

Sorting and filtering are read-only operations applied to the already-loaded table list; no additional database queries are required for each filter or sort change.
