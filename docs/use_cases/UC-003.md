# Use Case: Preview Table Data

## Overview

**Use Case ID:** UC-003
**Use Case Name:** Preview Table Data
**Primary Actor:** Developer
**Goal:** View the first rows of a table to quickly verify that data is present and correctly structured
**Status:** Draft

## Preconditions

- Schematic is embedded in the running Spring Boot application (UC-001 completed)
- The application's datasource is connected to a reachable database
- Developer has navigated to the table list and can see at least one table (UC-002)

## Main Success Scenario

1. Developer selects a table from the table list.
2. System retrieves the first rows of the selected table.
3. System displays the rows in a tabular format with column headers.
4. Developer reviews the data to verify its presence and structure.

## Alternative Flows

### A1: Table Is Empty

**Trigger:** The selected table contains no rows (step 2)
**Flow:**

1. System displays the column headers with no data rows and an appropriate empty-state message.
2. Use case ends.

### A2: Database Connection Lost During Preview

**Trigger:** The datasource cannot reach the database when row data is requested (step 2)
**Flow:**

1. System displays an error message indicating the data could not be retrieved.
2. Developer checks connectivity and retries.
3. Use case ends.

### A3: Table Has a Very Large Number of Columns

**Trigger:** The selected table has enough columns to exceed the viewport width (step 3)
**Flow:**

1. System renders the table with horizontal scrolling so all columns remain accessible.
2. Use case continues at step 4.

### A4: Query Execution Exceeds Performance Budget

**Trigger:** Retrieving the preview rows takes longer than the 2-second render target (step 2)
**Flow:**

1. System displays whatever results are available or shows a timeout message.
2. Use case ends.

## Postconditions

### Success Postconditions

- Developer can see the first rows and all column values of the selected table
- No data in the database is modified

### Failure Postconditions

- Row data is not displayed
- System shows an error or empty-state message
- No data in the database is modified

## Business Rules

### BR-001: Read-Only Operation

Previewing table data is a read-only operation; no database rows or structures are altered.

### BR-002: Row Limit

Only a limited number of rows (the first N rows) are fetched to keep the preview fast and avoid memory pressure on the host application.

### BR-003: Page Load Performance

The table data preview must complete its initial render within 2 seconds on a developer workstation connected to a local database.
